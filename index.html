Rukkit ships in two flavors — a classless stylesheet that styles plain semantic HTML, and a classed stylesheet with explicit .rk-* utilities. Same tokens, same look, your choice.

03 · Flavors

Classless or classed — same look, different ergonomics.

Use the classless flavor for greenfield projects and prose-heavy pages — write semantic HTML, get a styled page. Use the classed flavor when you need explicit variants and predictable utility names inside an existing app.

A · Card with a CTA
classless · zero classes
<article>
  <h5>Featured</h5>
  <h3>Hello, Rukkit</h3>
  <p>Drop semantic HTML, get a styled card.</p>
  <button>Open</button>
</article>
classed · explicit utilities
<article class="rk-card">
  <p class="rk-eyebrow">Featured</p>
  <h3 class="rk-subhead">Hello, Rukkit</h3>
  <p class="rk-body">Drop semantic HTML, get a styled card.</p>
  <button class="rk-btn">Open</button>
</article>
B · Form with primary submit
classless
<form>
  <label>Email</label>
  <input type="email" />
  <button type="submit">Subscribe</button>
  <button type="reset">Clear</button>
</form>
classed
<form>
  <label class="rk-field__label">Email</label>
  <input class="rk-input" />
  <button class="rk-btn rk-btn--primary">Subscribe</button>
  <button class="rk-btn rk-btn--ghost">Clear</button>
</form>

When to use which

Use classless when…
You're writing prose, docs, blog posts, or landing pages. You want the markup to stay readable. You like the look out of the box.
Use classed when…
You're building a complex app and need explicit variant control. You want intellisense on class names. You're mixing Rukkit with another framework.
Use both when…
Some apps benefit from classless prose regions inside a classed shell. The two flavors don't fight; [data-rukkit] scopes the classless rules.

Details / accordion

What's the difference between classless and classed?

The classless flavor styles semantic HTML directly — you don't write any class names. The classed flavor exposes explicit .rk-* classes with named variants. Same tokens, same look.

Can I use both flavors at once?

Yes. The classless rules are scoped to [data-rukkit], so they activate only where you opt in. The classed rules use a .rk-* prefix that won't collide.

Does it work without JavaScript?

Completely. The library is plain CSS. The playground page uses a few JS sprinkles for the sandbox, theme tweaker, and copy buttons — none of which is part of the library itself.

05 · Tokens

Every value is a CSS variable.

Override any --rk-* property to retheme — or grab the live theme tweaker in the bottom-right to play.

css · tokens
:root {
  /* Fonts */
  --rk-font-display: "Fraunces", Georgia, serif;
  --rk-font-body:    "Inter", system-ui, sans-serif;
  --rk-font-mono:    "JetBrains Mono", monospace;

  /* Surfaces */
  --rk-canvas:       #f4f0e6;
  --rk-surface:      #fbf8f1;
  --rk-ink:          #1a1714;

  /* Accent */
  --rk-accent:       #3a3aef;
  --rk-signal:       #cf4500;

  /* Radii */
  --rk-r-sm:         6px;
  --rk-r-md:         16px;
  --rk-r-pill:       9999px;
}
